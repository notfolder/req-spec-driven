# 開発タスクリスト

## 目的
本タスクリストは、上記の詳細設計に基づき、システムの最小実行可能製品（MVP）を実装するために必要な全ての作業ステップを定義します。

## タスク一覧

1. **[T-Setup] 環境構築と初期設定**
    *   **内容**: プロジェクト構造のセットアップ (`src/`, `utils/`) および依存ライブラリのインストール。
    *   **完了条件**: `pip install -r requirements.txt` が成功し、全てのディレクトリが作成された状態であること。
    *   **バリデーションタスク**: `ls src/` を実行し、主要ファイルが存在することを確認する。

2. **[T-Logger] ログ管理モジュールの実装と共通化** (高優先度)
    *   **内容**: `src/logger_manager.py` を作成し、シングルトンパターンを用いて基本的なログ記録ロジック (`log_event`) を実装する。
    *   **完了条件**: Pythonスクリプトからこのモジュールを呼び出し、手動で複数の異なるイベントタイプ（SUCCESS, ERROR）のメッセージがメモリに追記され、`get_history()` で取得できること。
    *   **バリデーションタスク**: `logger_manager.py` からログオブジェクトを取得し、タイムスタンプと必須フィールドが存在するか確認する。

3. **[T-DataProcessor] データ取り込みとバリデーション機能の実装** (高優先度)
    *   **内容**: `src/data_processor.py` を作成。CSV読み込み(`load_data`)、および共通化された数値データ検証ロジック (`validate_columns`) を実装する。
    *   **完了条件**: ダミーのテスト用CSVファイルを渡した際に、バリデーションが成功し、Pandas DataFrameオブジェクトとしてデータが返却されること。失敗時には適切なエラーメッセージを返すこと。
    *   **バリデーションタスク**: 意図的に非数値を含むカラムを持つダミーデータを渡し、`validate_columns` が正しく処理をブロックすることを確認する。

4. **[T-Analyzer] 統計解析エンジンの実装** (高優先度)
    *   **内容**: `src/analysis_engine.py` を作成し、Q-Qプロット算出ロジックと基本的な記述統計量計算ロジックを実装する。この際、共通の正規化処理は内部共通モジュールを利用する。
    *   **完了条件**: ダミーデータセット（数値のみ）に対して、手動で算出した結果と比較して差異がないこと。描画に必要な生のデータ配列が返却されること。
    *   **バリデーションタスク**: データのサンプリング数が少ない場合に、適切に警告を出すロジックが機能することを確認する。

5. **[T-Streamlit] メインアプリのオーケストレーション構築** (最高優先度)
    *   **内容**: `src/streamlit_app.py` を作成し、データフロー（アップロード $\to$ 選択 $\to$ 解析）を制御するUIロジック全体を実装する。各ステップで他のモジュール (`DataProcessor`, `Analyzer`, `LoggerManager`) を呼び出す順序を確立する。
    *   **完了条件**: アプリケーションが単体で動作し、全ての画面遷移とデータフローがシミュレーション通りに機能すること。
    *   **バリデーションタスク**: 各ステップの実行後、`LoggerManager.get_history()` を表示し、ログの状態遷移（UNPROCESSED $\to$ PROCESSING $\to$ COMPLETED）が正しいか確認する。

6. **[T-Test] E2Eテスト環境構築と実装** (高優先度)
    *   **内容**: `e2e/` ディレクトリを作成し、Playwrightを使用した全てのシナリオ（基本フロー, エラーハンドリング, 比較対象の変更）のテストコードを記述する。また、`docker-compose.yml`に`test_playwright`サービスとプロファイル設定を組み込む。
    *   **完了条件**: 上記3つのE2Eテストケース全てがPlaywrightで成功すること（`npx playwright test`実行時）。
    *   **バリデーションタスク**: `docker-compose.yml` を確認し、`profiles: test` が正しく設定され、必要な環境変数が定義されていることを検証する。

7. **[T-Finalize] ドキュメント化と最終レビュー** (最高優先度)
    *   **内容**: 設計書の内容を基にコードコメントの追加、README.md の作成（起動・操作説明）、および全てのモジュールを統合し、コードの品質と整合性を最終確認する。
    *   **完了条件**: 全てのタスクが「completed」ステータスとなり、動作検証が成功した状態であること。

## 実行指示
上記のリストに従って、「T-Setup」から順に作業を進めます。まずは [T-Logger] の実装から開始します。
